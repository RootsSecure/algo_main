# Acceptance Cases

## Case 1: New owner onboards a plot
Expected result: the property record exists, zones can be added, and devices can be linked.

## Case 2: Heartbeat is recorded
Expected result: device health updates and a site health check is stored.

## Case 3: Tamper alert is ingested
Expected result: severity is critical, an incident is linked, and notification logs are created.

## Case 4: Incident is verified and dispatched
Expected result: incident status progresses and a partner dispatch record exists.

## Case 5: Dispatch closes the incident
Expected result: incident resolves, closure notes are stored, and export summary reflects dispatch state.

## Case 6: Delegate access
Expected result: invited delegate can access assigned property after accepting the invite.
