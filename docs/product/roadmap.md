# Roadmap

## Phase 1: Backend foundation

- Authentication and role model
- Property, zone, device, alert, incident, partner, and evidence entities
- Health and incident APIs
- Project docs index and repository documentation baseline

## Phase 2: Operational maturity

- Real notification provider adapters
- Field-service SLAs and dispatch analytics
- Incident queue UI for operations users
- Better verification tooling and evidence packaging

## Phase 3: Product expansion

- Owner-facing mobile application
- Caretaker workflows and check-in tasks
- Vendor-specific device adapters
- Scheduled health reports and policy automation

## Phase 4: Scale and compliance

- PostgreSQL production posture
- Media lifecycle automation
- Stronger audit and retention controls
- Regional operations dashboards and multi-property portfolios
